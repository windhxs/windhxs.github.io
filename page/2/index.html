本文介绍tensorboard工具的使用

tensorboard 通常用于模型和参数的可视化，学习tensorboard对于深度神经网络可以用更好的理解

本文基于tensorboard官网的样例和代码

遇到的问题小结（MAC OS)

使用anaconda安装一些库时会安装最新版本，可能会和系统有些冲突的存在

protobuf

默认安装版本是3.11.2，需要降到3.8.0

如何使用tensorboard

在model的fit()方法传递callbacks参数

查看fit()方法的api

参数 + batch_size 一次训练的样本数，增大则训练速度变快，但内存消耗也随之增加 + epochs 全部训练样本被训练的次数 + validation_split 0-1之间的小数，表示划分多少训练样本为验证集 + validation_data 验证样本，会覆盖validation_split，对于损失函数和精确度进行计算，但不会用于训练中 + shuffle 是否对于训练样本进行随机排列 + callbacks keras.callbacks.Callback类的对象，可以是tf.keras.callbacks.TensorBoard对象

fit()返回值为History对象。

History有两个属性，分别是history(dictionary)和epoch(训练的epochs)

定义一个tensorboard

参数 + log_dir 存储tensorboard日志文件的路径 + histogram_freq 计算激活值和权重直方图的频率 + write_graph 是否查看计算图 + update_freq 更新频率，‘batch'，‘epoch'或者是整数，即整数个batch

tf.keras.layers.Conv1D

该层的作用是对于输入进行一维卷积

__init__代码

参数

  • filters 卷积核的个数
  • kernel_size 整数，卷积核的大小
  • stride 整数，卷积核每次移动的长度
  • padding 'valid'、‘causal’、‘same’
  • data_form channels_last/channels_first
  • dilation_rate 也是卷积核移动的长度，和stride有冲突，一般不考虑
  • activation 激活函数
  • use_bias 是否在卷积操作后对结果加上一个偏置
  • kernel_initializer 对卷积核进行初始化
  • bias_initializer 对偏置进行初始化
  • kernel_regularizer 对卷积核进行正则化
  • bias_regularizer 对偏置进行正则化
  • activity_regularizer 对激活函数进行正则化
  • kernel_constraint 对卷积核进行限制
  • bias_constraint 对偏置进行限制

输入

三维张量(batch_size, steps, input_dim)

输出

三维张量(batch_size, new_steps, filters)

  • bias_regularizer 对偏置进行正则化
  • activity_regularizer 对激活函数进行正则化
  • kernel_constraint 对卷积核进行限制
  • bias_constraint 对偏置进行限制

tf.keras.activations.elu

elu函数表达式如下 \[ \begin{equation} f(x) = \begin{cases} x & \text{ if } x>=0 \\\\ \alpha*(e^x-1) & \text{ if } x<0 \end{cases} \end{equation} \] 超参数只有\(\alpha\),查看tf.keras中关于elu的代码如下

参数： - x 输入的张量 - alpha 一个常数

返回值:elu激活函数

tf.keras.activations.exponential

exponential函数表达式如下 \[ f(x) = e^x \] 没有超参数，查看tf.keras中关于exponential的代码如下

参数 - x 输入的张量

返回值 - exp激活函数

tf.keras.activations.linear

linear函数表达式如下 \[ f(x) = x \] 没有超参数，查看tf.keras中关于exponential的代码如下

参数 - x 输入的张量

返回值 - linear激活函数,该激活函数并不对输入产生非线性变换

tf.keras.activations.rule

rule函数表达式如下 \[ f(x) = max(0,x) \] ,而leaky rule表达式为 \[ f(x) = max(0.1x,x) \] 在tf.keras中，对于rule有了更细致的定义，查看关于rule的代码如下

翻译一下就是 \[ f(x) = \begin{cases} max\underline{}value & \text{if} x >= max\underline{}value\\\\ x & \text{ if } x < max\underline{}alue\ \&\ x>=threshold \\\\ \alpha*x & \text{ if } x <max\underline{}value\ \&\ x <threshold \end{cases} \] 默认情况下就是普通的rule函数，即f(x)=max(0,x)
参数 - x 输入的张量 - alpha 左半部分的斜率 - max_value 整个函数能产生的最大值 - threshold 两个部分的分界线

返回值 - rule 激活函数

tf.keras.activations.selu

selu函数表达式如下 \[ f(x) = \begin{cases} scale*x & \text{ if } x>=0 \\\\ scale *\alpha * (e^x -1)& \text{ if } x<0 \end{cases} \] 其中\(scale\)和\(\alpha\)已经是确定的 - \(scale\) : 1.05070098 - \(\alpha\) : 1.67326324
tf.keras中关于selu的代码如下

参数 - x 输入的张量

返回值 - selu激活函数

selu的目的主要是调整\(x>=0\)时函数的斜率

参数 - x 输入的张量 - axis 正数，表示softmax操作的坐标轴

返回值 - 张量
