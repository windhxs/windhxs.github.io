tf.keras.metrics.FalseNegatives

有threshold参数

tf.keras.metrics.FalsePositives

同样有threshold参数

同样有threshold参数

tf.keras.metrics.Precision

\[ Precision = \frac{TP}{TP + FP} \]

tf.keras.metrics.Recall

\[ Recall = \frac{TP}{TP + FN} \]

tf.keras.metrics.AUC

谈AUC，就不得不说ROC(Receiver Operation Characteristic)

定义 \[ TPR = \frac{TP}{TP + FN} = Recall \] \[ FPR = \frac{FP}{FP + TN} \] ROC曲线的纵轴是TPR,横轴是FPR

AUC 就是该曲线和X轴的面积

参数

  • num_threholds 计算的阈值数量
  • curve 曲线类型
    • 'ROC'
    • 'PR Precision-Recall曲线
  • summation_method 面积累加的方法
    • 'interpolation' 中间点的值作为高
    • 'minoring' 值小的那个点作为高
    • 'majoring' 值大的那个点作为高
  • name 名称
  • dtype 返回值的数据类型
  • threholds 以列表的形式，自定义阈值
  • multi_label 是否有多个类别
  • label_weights 类别的权重

如何画ROC曲线

假设样本有m个正例，n个反例 - 先将阈值调至最大，即则所有预测均为反例，此时TPR = FPR = 0 ,在坐标原点标记一个点 - 依次将阈值改变为设定的值，分别计算TPR，和FPR，在坐标轴中标记，相邻的点连成线即可

图像的矩

对于一个二元函数\(f(x,y)\),其\((p+q)\)阶矩定义为： \[ M_{p q}=\int_{-\infty}^{\infty} \int_{-\infty}^{\infty} x^{p} y^{q} f(x, y) d x d y \] 对于一个图像(灰度图)，\(I(x,y)\)表示\((x,y)\)处的像素值，将图像的像素值考虑为概率密度函数，则其矩为 \[ M_{i j}=\sum_{x} \sum_{y} x^{i} y^{j} I(x, y) \]

图像矩有一些应用 + 在二进制图像中，图像的区域是\(M_{0 0}\) + 图像的质心为\(\{\bar{x}, \bar{y}\}=\left\{\frac{M_{10}}{M_{00}}, \frac{M_{01}}{M_{00}}\right\}\)

中心距

定义为 \[ \mu_{p q}=\int_{-\infty}^{\infty} \int_{-\infty}^{\infty}(x-\bar{x})^{p}(y-\bar{y})^{q} f(x, y) d x d y \] 其中 \[ \bar{x} = \frac{M_{1 0}}{M_{0 0}}\ , \ \bar{y} = \frac{M_{0 1}}{M_{0 0}} \]

若\(f(x,y)\)是图片的话，则 \[ \mu_{p q} =\sum_{x} \sum_{y}(x-\bar{x})^{p}(y-\bar{y})^{q} I(x, y) \]

可以得到： \[ \begin{array}{l} {\mu_{00}=M_{00}} \\ {\mu_{01}=0} \\ {\mu_{10}=0} \\ {\mu_{11}=M_{11}-\bar{x} M_{01}=M_{11}-\bar{y} M_{10}} \\ {\mu_{20}=M_{20}-\bar{x} M_{10}} \\ {\mu_{21}=M_{21}-\bar{x} M_{10}} \\ {\mu_{21}=M_{21}-2 \bar{x} M_{11}-\bar{y} M_{20}+2 \bar{x}^{2} M_{10}} \\ {\mu_{12}=M_{30}-3 \bar{x} M_{20}+2 \bar{x}^{2} M_{10}} \\ {\mu_{03}=M_{03}-3 \bar{y} M_{02}+2 \bar{y}^{2} M_{01}} \end{array} \] 同时可得,\(\mu_{pq}\)和\(M_{mn}\)的关系是 \[ \mu_{p q}=\sum_{m}^{p} \sum_{n}^{q}\left(\begin{array}{l} {p} \\ {m} \end{array}\right)\left(\begin{array}{c} {q} \\ {n} \end{array}\right)(-\bar{x})^{(p-m)}(-\bar{y})^{(q-n)} M_{m n} \]

tf.keras.optimizers.SGD

随机梯度下降

最原始的\(SGD\): \[ x_{t+1}=x_{t}-\alpha \nabla f\left(x_{t}\right) \]

\(SGD+Momentum\):

\[ \begin{aligned} &v_{t+1}=\rho v_{t}+\nabla f\left(x_{t}\right)\\ &x_{t+1}=x_{t}-\alpha v_{t+1} \end{aligned} \]

\(Nesterov\ Momentum\):

\[ \begin{aligned} &v_{t+1}=\rho v_{t}-\alpha \nabla f(x_{t}+\rho v_{t})\\ &x_{t+1}=x_{t}+v_{t+1} \end{aligned} \]

这篇解说不错

参数

  • learning_rate 学习率
  • momentum 动量值
  • nesterov 是否应用Nesterov Momentum
  • name 名称
  • **kwargs 暂时不用管
    • clipnorm 暂时不管
    • clipvalue 暂时不管
    • lr 就是learning_rate
    • decay 暂时不管

tf.keras.optimizers.Adagrad

\[ \begin{aligned} & dx = \nabla f(x_t) \\ & grad += dx * dx \\ &x_{t+1} = x_{t} - \frac{\alpha * grad } { \sqrt{grad + \epsilon }} \end{aligned} \] 其中\(\epsilon\) 是一个很小的数，为了避免除零

参数

  • learning_rate 学习率
  • initial_accumulator_value \(grad\)初始值
  • epsilon 即\(\epsilon\)
  • name 名称
  • **kwargs 暂时不用管
    • clipnorm 暂时不管
    • clipvalue 暂时不管
    • lr 就是learning_rate
    • decay 暂时不管

tf.keras.optimizers.RMSprop

\(RMSprop\)是对\(Adagrad\)的小小改动

\[ \begin{aligned} & dx = \nabla f(x_t) \\ & grad = \rho * grad + (1 - \rho) * dx * dx \\ &x_{t+1}= x_{t} - \frac{\alpha * grad } { \sqrt{grad + \epsilon }} \end{aligned} \]

\(RMSprop + Momentum\) \[ \begin{aligned} & dx = \nabla f(x_t) \\ & grad = \rho * grad + (1 - \rho) * dx * dx \\ &x_{t+1}= (1 + momentum) * x_{t} - \frac{\alpha * grad } { \sqrt{grad + \epsilon }} \end{aligned} \]

参数

  • learning_rate 学习率
  • rho 即\(\rho\)
  • momentum 动量
  • epsilon 即\(\epsilon\)
  • centered 是否进行归一化
  • name 名称
  • **kwargs 暂时不用管
    • clipnorm 暂时不管
    • clipvalue 暂时不管
    • lr 就是learning_rate
    • decay 暂时不管

tf.keras.optimizers.Adam

\[ \begin{aligned} & dx = \nabla f(x_t) \\ & mom_{first} = beta_1 * mom_{first} + (1- beta_1) * dx \\ & mom_{second} = beta_2 * mom_{second} + (1- beta_2) * dx * dx\\ &x_{t+1}= x_{t} - \frac{\alpha * mom_{first}} { \sqrt{mom_{second} + \epsilon }} \end{aligned} \]

参数

  • learning_rate 学习率
  • beta_1 超参数
  • beta_2 超参数
  • epsilon 即\(\epsilon\)
  • amsgrad 是否用\(AMSGrad\)
  • name 名称
  • **kwargs 暂时不用管
    • clipnorm 暂时不管
    • clipvalue 暂时不管
    • lr 就是learning_rate
    • decay 暂时不管

所谓模型的保存，其实就是保存模型中参数(而不是超参数)的值。

以checkpoints保存

在模型训练过程中，即调用用fit()时，设置checkpoints参数为一个tf.keras.callbacks.ModelCheckpoint类的对象，就可以在一边训练时，一边保存参数

tf.keras.callbacks.ModelCheckpoint

其__init__为：

参数

  • filepaht 保存的文件名
  • monitor 被监测的数据
  • verbose 详细信息模式，即是否输出相关信息
  • save_best_only 是否保存最好的
  • save_weights_only 是否只保存权重
  • mode save_best_only = True时才有效
    • auto 自动判断
    • min 取最小值，一般当monitor为val_loss时使用
    • max 取最大值，一般当monitor为val_acc时使用
  • save_freq 每个Checkpoint的间隔单位
  • period 几个间隔单位

使用tf.keras.callbacks.ModelCheckpoint

和tensorboard的使用方法类似，定义好对象后，在fit()中的callbacallbacks参数设置为该对象即可。

  • byname 加载权重是是根据名称还是拓扑排序
  • skip_mismatch 当某层权重数量不匹配是是否跳过

保存整个模型

参数
