tensorflow模型的保存

Posted on 2020-02-08

所谓模型的保存，其实就是保存模型中参数(而不是超参数)的值。

以checkpoints保存

在模型训练过程中，即调用用fit()时，设置checkpoints参数为一个tf.keras.callbacks.ModelCheckpoint类的对象，就可以在一边训练时，一边保存参数

tf.keras.callbacks.ModelCheckpoint

其__init__为：

参数

  • filepaht 保存的文件名
  • monitor 被监测的数据
  • verbose 详细信息模式，即是否输出相关信息
  • save_best_only 是否保存最好的
  • save_weights_only 是否只保存权重
  • mode save_best_only = True时才有效
    • auto 自动判断
    • min 取最小值，一般当monitor为val_loss时使用
    • max 取最大值，一般当monitor为val_acc时使用
  • save_freq 每个Checkpoint的间隔单位
  • period 几个间隔单位

使用tf.keras.callbacks.ModelCheckpoint

和tensorboard的使用方法类似，定义好对象后，在fit()中的callbacallbacks参数设置为该对象即可。

加载已保存的权重

参数

  • filepath 文件路径
  • byname 加载权重是是根据名称还是拓扑排序
  • skip_mismatch 当某层权重数量不匹配是是否跳过

手动保存权重

参数

  • filepath 路径
  • overwrite 是否覆盖同名文件
  • save_format 保存的格式
    • tf tensorflow格式
    • h5 HDF5格式

保存整个模型

参数

  • filepath 路径
  • overwrite 是否覆盖
  • include_optimizer 是否保存优化器
  • save_format 保存的格式
    • tf tensorflow格式
    • h5 HDF5格式
  • signature 暂时不用管
  • options 暂时不用管

加载模型

参数

  • filepath 路径
  • custom_objects 暂时不用管
  • compile 加载后是否进行编译

返回值

keras model对象

自适应阈值

在学习记录#5的时候，记录了cv.threshold可以将灰度图经过一个阈值的处理，输出对应的图像，但是手工设置thres,既困难又不方便，因此有了自适应阈值这种神奇的东西

参数

  • src 原图像
  • maxValue 设置的最大值
  • addtiveMehtod 自适应的方式
    • cv.ADAPTIVE_THRESH_MEAN_C 阈值为bockSize*blockSize区域所有像素的平均值减去C
    • cv.ADAPTIVE_THRESH_GAUSSIAN_C 阈值为bockSize*blockSize区域经过高斯滤波器后的值减去C
  • thresholdType 只有两种
    • cv.THRESH_BINARY maxval或者0，以thresh分界
    • cv.THRESH_BINARY_INV 0或maxval，以thresh分界
  • bolckSize 像素相邻域的大小
  • C 要减去的常数
  • dst 返回值，生成的图像

返回值

dst 生成的图像

图像二值化 Otsu's Binarization

在cv.threshold()中，将参数设为cv.THRESH_BINARY+cv.THRESH_OTSU即可应用Otsu算法，该函数返回两个值，分别是retval,dst ，而retval就是获得的阈值

HSV 色彩空间

在cv.cvtColor中，当参数 是cv.COLOR_BGR2HSV时，可以将bgr色彩空间转化为HSV空间。

通常情况下，HSV的各项取值分别是，H(Hue ,色相) [0,360）,S(Saturation ,饱和度) [0，100）,V(Value ,明度) [0,100)

而在OpenCV中，对三个值有进行了一定的修改，起取值范围分别是， H [0,180],S [0,256),V[0,256)

即H' = H / 2 , S'(/V) = S(/V) * 1.0 / 100 * 256

图片的缩放

参数

  • src 原图片
  • dsize 输出图片的形状
  • dst 输出的图片
  • fx 水平方向的放缩因子
  • fy 垂直方向的放缩因子
  • interpolation 插值方法
    • cv.INTER_NEAREST 最近邻插值
    • cv.INTER_LINEAR 双线性插值
    • cv.INTER_CUBIC 双三次插值
    • cv.INTER_AREA 图片缩小时对区域进行重采样，放大时和最近邻插值差不多
    • cv.INTER_LANCZOS4 Lanczos插值

注

dsize和fx、fy不要同时设置

返回值

缩放后的图片 ## 图片的移动

参数

  • src 原图像
  • M 2×3的矩阵
  • dsize 输出图像的大小
  • dst 输出的图像
  • flags 插值方法
  • borderMode 边缘类型
    • cv.BORDER_REPLICATE 填充最外层的像素值
    • cv.BORDER_CONSTANT 填充常数
    • cv.BORDER_REFLECT 翻转
    • cv.BORDER_WRAP 平移
    • cv.BORDER_REFLECT_101 除了最边上的像素翻转
    • cv.BORDER_TRANSPARENT
  • borderValue 当borderMode=cv.BORDER_CONSTANT时设置的值

dst移动后的图像

ImageDataGenerator类

Posted on 2020-01-31

本文自我总结ImageDataGenerator类

ImageDataGenerator通常用于创造图像数据，并且对数据进行预处理

__init__方法

源码

参数

  • featurewise_center 是否将整个数据集的均值变为0
  • samplewise_center 是否将每张图片均值变为0
  • featurewise_std_normalization 是否除以整个数据集的标准差，即标准化
  • samplewise_std_normalization 是否除以自身的标准差
  • zca_whitening 是否进行zca白化
  • zca_epsilon zca白化过程中为了避免出现除0而分母需要增加的一个很小的数
  • rotation_range 图片随机转动的角度的范围，即图片在0到该范围内随机旋转
  • width_shift_range 图片水平偏移的幅度
  • height_shift_range 图片垂直偏移的幅度
  • shear_range 错切变换
  • zoom_range 随机缩放的幅度
  • channel_shift_range 随机通道偏移的幅度
  • fill_mode ‘constant’，‘nearest’，‘reflect’或‘wrap’,当变换后的图片产生了空档该如何填充
  • cval fill_mode=constant时的填充值
  • horizontal_flip 进行随机水平翻转
  • vertical_flip 进行随机竖直翻转
  • rescale 放缩因子，将该因子乘上数据，该操作会在所有的变换之前进行
  • preprocessing_function 被应用于每个输入的函数，最后进行，该函数必须接受一张图片，并返回相同形状的np.array
  • data_format ‘channels_first’或者‘channels_last’
  • validation_split 验证数据集所占比例
  • dtype 生成的数据类型

返回值

ImageDataGenerator类对象

apply_transform方法

参数

  • x 三维张量，单张图片
  • transform_parameters 字典，主要参数有
    • 'theta': Float. Rotation angle in degrees.
    • 'tx': Float. Shift in the x direction.
    • 'ty': Float. Shift in the y direction.
    • 'shear': Float. Shear angle in degrees.
    • 'zx': Float. Zoom in the x direction.
    • 'zy': Float. Zoom in the y direction.
    • 'flip_horizontal': Boolean. Horizontal flip.
    • 'flip_vertical': Boolean. Vertical flip.
    • 'channel_shift_intencity': Float. Channel shift intensity.
    • 'brightness': Float. Brightness shift intensity.

返回值

和输入相同形状的张量

fit方法

对样本集进行处理，当‘featurewise_center’、'featurewise_std_normalization'、'zca_whitening'三者至少有一个设置为True时才能使用 ### 参数 + x 样本集，秩应该为 4。对于灰度图，通道轴的值应该为1 + augment 是否使用随机样本扩张 + rounds 随机扩张的次数 + seed 随机数种子

返回值

无

flow方法

获取数据和标签，生成一批扩张的数据 ### 参数 + x 数据集，秩为4 + y 标签 + batch_size 一批的数据量 + shuffle 是否将数据随机重排 + sample_weight 样本权重 + seed 随机数种子 + save_to_dir 指定要保存的正在生成的增强图片的目录 + save_prefix 保存图片的前缀 + save_format 保存的图片的类型 + subset ’training'或者'training'，只有在构造函数中设置了'validation_split'参数

返回值

一个(x, y)元祖的Iterator

  • batch_size 一批数据的大小
  • shuffle 是否将数据随机重排
